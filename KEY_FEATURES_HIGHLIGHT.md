### 6. 🛡️ 故障转移机制

#### 备用AI服务
当Dify服务不可用时，系统自动切换到本地AI：

```java
public String getChatResponse(String message, String userId) {
    try {
        // 尝试调用外部AI服务
        return callExternalAI(message, userId);
    } catch (Exception e) {
        log.warn("外部AI服务调用失败，使用备用响应");
        // 自动切换到备用AI
        return getBackupResponse(message, userId);
    }
}
```

#### 备用服务特点
- ==**智能关键词匹配**==：准确理解用户意图
- ==**完整功能覆盖**==：房间查询、预订、活动推荐
- ==**无缝切换**==：用户无感知的服务降级
- ==**快速响应**==：< 1秒响应时间
